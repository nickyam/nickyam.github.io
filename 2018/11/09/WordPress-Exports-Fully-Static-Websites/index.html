推荐WordPress插件导出静态网站

Nov 9, 2018 | 科技 | 493 字 | 1 分钟

之前建站初入门的时候，一般都是Wordpress，一有问题网上基本都能找到解决方法，而且主题丰富且漂亮。但是因为Wordpress是动态网站页面，性能消耗较大，而作为个人博客，需要的只是一个完全静态的网站。

什么是静态网站？完全静态的网站只有html文件，对服务器资源消耗极低，可以免费托管在 Github Page 等。对于学生等个人站长而言，运行成本非常低。此外，静态网站安全性很好，几乎没有什么可攻击的漏洞，即使恶意访问量很大，静态网站也可以很好地支撑。所以不少站长选择从 Wordpress 转移到 静态网站。

那么，如何将一个 Wordpress 网站完全静态化呢？其实 Wordpress 给我们提供了很好的工具。

  • 文章页静态化：在Wordpress“设置”-“固定连接”里，将固定链接设置为“自定义结构”，文章的标题结尾增加.html。此时，目录和标签还是动态页面。

  • 关闭评论功能，在“设置”-“讨论”里，选择“自动关闭发布1天后的文章上的评论功能”。

  • 安装Simply Static插件生成静态网站。如果文章数量上千，则容易报错，原因是一般的网站服务器会在php上设置了超时和内存限制。可在本地搭建php环境，修改php超时时间和内存大小，例如对于Windows系统来说，可以选择安装宝塔面板，快速配置WAMP环境，修改环境里php超时时间和内存大小即可。

  • 采用 WP2Static 插件。如果你对生成静态网站的时间要求较高，则建议采用该插件，并修改Crawl Increment 的数量为100以上的数值。

如果你有更好的插件或方法，也请通过 nickyam@163.com 告诉我，谢谢！
